# 测试回归用例 · 期望路由

> 以下测试用例用于验证 Product Route Skill 的路由判断是否稳定。每个用例定义了产品想法、期望路由、必须包含的要素和不得出现的错误。

---

## T001 · 二维码生成器

**输入：** 做一个二维码生成器，用户可以输入文本、URL、联系方式，生成可定制的二维码图片。

**期望主路由：** R01（入口工具）

**不得：**
- 推荐高价订阅 SaaS
- 路由为 R02（货架服务）
- 建议做用户系统、付费系统

**必须：**
- 说明付费意愿弱
- 建议靠 SEO / 免费分发获客

---

## T002 · 完整视频托管平台

**输入：** 做一个完整的视频托管平台，让创作者上传视频、自动转码、播放、嵌入分享。

**期望主路由：** R07（高风险基础设施，暂缓）

**期望副路由：** R04（第三方封装 / 集成层）

**不得：**
- 建议从零做完整视频托管 MVP
- 忽略存储、带宽、转码、版权风险
- 路由为 R01 或 R02

**必须：**
- 命中风险门槛 G05（视频、存储、带宽风险）
- 建议窄化为 Vimeo/Mux/Wistia 封装管理面板

---

## T003 · 政策页托管

**输入：** 给独立开发者和小型 SaaS 提供政策页面（隐私政策、服务条款）托管服务。

**期望主路由：** R02（货架服务）

**期望副路由：** R01（入口工具）

**必须：**
- 说明这不是法律意见
- 说明责任风险 S09 = 3（中等）
- 说明不承诺合规兜底

**不得：**
- 路由为 R05（销售型服务）
- 承诺法律合规保证

---

## T004 · 完整邮件发送平台

**输入：** 做一个完整的邮件发送平台，用户可以发送事务邮件、营销邮件、批量邮件。

**期望主路由：** R07（高风险基础设施，暂缓）

**期望副路由：** R04（第三方封装 / 集成层）

**不得：**
- 忽略送达率、退信、投诉、域名信誉风险
- 忽略 SPF / DKIM / DMARC 配置要求
- 建议从零造 SMTP 服务器

**必须：**
- 命中风险门槛 G03（邮件送达风险）
- 建议使用 Resend/SendGrid/SES 封装

---

## T005 · 轻量客服消息收件箱

**输入：** 做一个轻量客服消息收件箱，统一查看邮件、聊天、社交媒体消息。

**期望主路由：** R02（货架服务）或 R04（第三方封装）

**不得：**
- 直接建议做完整 Intercom / Zendesk 替代品
- 忽略客服系统 SLA 和操作负担
- 路由为 R07（除非用户明确要做完整客服平台）

**必须：**
- 说明需要封装的第三方渠道
- 说明 MVP 应该窄化为消息聚合查看

---

## T006 · AI 图片生成 API

**输入：** 做一个 AI 图片生成 SaaS，用户可以输入 prompt 生成图片，按月订阅。

**期望主路由：** R04（第三方封装 / 集成层）或 R02（货架服务）

**不得：**
- 忽略底层 AI API 成本和可用性风险
- 建议从零训练模型

**必须：**
- 说明依赖的第三方 AI API
- 说明成本结构和定价风险
- 说明维护成本 S10 >= 3（需要持续关注 API 变更）

---

## T007 · 内部 CI/CD 面板

**输入：** 做一个小团队内部用的 CI/CD 管理面板，不打算对外销售。

**期望主路由：** R03（内部基础设施）

**不得：**
- 路由为 R02（货架服务）
- 建议立即对外销售

**必须：**
- 说明复用价值 S11 高
- 说明获客路径不清晰（因为不对外）
- 建议先满足内部需求

---

## T009 · AI Twitter 内容摘要工具

**输入：** 做一个 AI 工具，自动抓取和摘要 Twitter/X 上的热门长线程、列表和书签内容。

**期望主路由：** R06（内容或社区产品）

**不得：**
- 路由为 R02（货架服务 / SaaS）
- 建议先建 Web Dashboard 和用户系统

**必须：**
- 说明核心价值是内容输出，不是工具功能
- 说明先做内容分发循环，再做自动化
- MVP 不包含用户系统、付费、Dashboard

---

## T008 · 个人博客系统

**输入：** 做一个个人博客系统，支持 Markdown 写作、自定义主题、RSS。

**期望主路由：** R08（只适合做功能，不适合独立产品）

**期望副路由：** R01（入口工具）

**不得：**
- 建议作为独立 SaaS 创业
- 忽略成熟竞品（WordPress、Ghost、Hugo、Hexo 等）

**必须：**
- 说明替代方案极成熟（S04 >= 4）
- 说明付费意愿弱（S05 <= 2）
- 说明可以作为学习项目或开源模块
