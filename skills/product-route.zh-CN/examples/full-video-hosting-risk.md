# 示例评估：完整视频托管平台

## 产品想法

做一个完整的视频托管平台，让创作者上传视频、自动转码、播放、嵌入分享，类似 YouTube 或 Vimeo。面向独立创作者和小型教育内容团队。

---

## 目标用户

- 独立内容创作者
- 小型在线教育团队
- 需要视频嵌入的 SaaS 产品

---

## 触发场景

- 创作者需要一个地方托管视频并嵌入到自己的网站
- 课程平台需要视频播放和管理
- 不想依赖 YouTube 的品牌和广告

## 风险门槛

### 命中的风险门槛

| 门槛 ID | 名称 | 触发原因 | 窄化建议 |
|---|---|---|---|
| G05 | 视频、存储、带宽风险 | 涉及视频托管、转码、大带宽 | 使用 Mux/Vimeo/Wistia API 做管理面板，不做底层托管 |
| G08 | 部署与基础设施 SLA 风险 | 需要高可用性和 CDN 分发 | 使用成熟 CDN 和云平台，不自己管理基础设施 |
| G04 | 隐私与合规风险 | 涉及用户上传内容，可能有个人信息 | 明确内容审核政策，使用第三方合规存储 |

| G03 | 邮件送达风险 | 非核心邮件发送 |

## 路由判断

**主路由：** R07 — 高风险基础设施，暂缓

**副路由：** R04 — 第三方封装 / 集成层

**命中的路由规则：**
- S09 >= 4 且 S08 >= 4 → R07 高风险基础设施，暂缓（规则 1）

**为什么不是其它路线：**
- R01：交付太复杂，不能做轻量入口工具
- R02：责任和交付风险太高，不适合自助 SaaS
- R04：如果窄化为 Mux/Vimeo 管理面板，可以做 R04 → 副路由
- R05：可以作为销售型服务，但不解决核心风险
- R08：视频托管本身有其独立价值，不是「只适合做功能」

---

## MVP 边界

**如果窄化为 R04 方案，第一版只做什么：**
1. Vimeo / Mux / Wistia 嵌入管理面板
2. 视频课程目录和播放列表
3. 嵌入代码生成器

**明确不做什么：**
1. 不做视频上传和转码
2. 不做 CDN 分发管理
3. 不做内容审核系统

**依赖第三方服务：**
| 服务 | 用途 | 替代方案 |
|---|---|---|
| Mux / Vimeo / Wistia | 视频托管和转码 | Bunny.net Stream |
| Cloudflare | CDN | Fastly |

**不承诺：**
1. 不承诺视频托管 SLA（由第三方负责）
2. 不承诺内容合规（用户自行管理上传内容）

---

## 不做清单

1. ❌ 不做视频上传和转码系统
2. ❌ 不做 CDN 分发基础设施
3. ❌ 不做内容审核 AI 系统
4. ❌ 不做实时直播
5. ❌ 不做广告投放系统
6. ❌ 不做视频分析（由 Mux 等底层服务提供）
7. ❌ 不做创作者变现和支付

## 最低成本验证动作

### 动作 1：Mux API 接入测试

**目标：** 验证封装 Mux API 的可行性和成本

**具体步骤：**
1. 注册 Mux 账号，阅读 API 文档
2. 用 Mux API 上传 3 个测试视频
3. 记录 API 调用流程、限制、定价
4. 评估封装层的开发工作量

**预计时间：** 1 天

---

### 动作 2：竞品研究和管理面板需求访谈

**目标：** 确认「视频管理面板」是否有独立需求

**具体步骤：**
1. 搜索 "video management dashboard for creators" "video course hosting"
2. 分析 Uscreen、Teachable、Kajabi 等课程平台的视频管理功能
3. 在创作者社区发帖询问「你是否需要一个更好的视频管理面板，还是直接使用 Vimeo/Wistia 就够了？」
4. 访谈 3 位内容创作者，问：你现在怎么管理你的视频？

**预计时间：** 2 天

---

### 动作 3：无代码原型验证

**目标：** 用最小成本验证视频管理面板的需求

**具体步骤：**
1. 用 Notion / Airtable 做一个视频管理模板（嵌入 Vimeo/Mux 视频链接）
2. 包含：视频列表、状态、嵌入代码、播放统计
3. 分享给 3-5 位潜在用户试用
4. 观察他们的使用反馈

**预计时间：** 1 天

---

## 外部证据缺口

- 还缺内容创作者的视频管理具体痛点
