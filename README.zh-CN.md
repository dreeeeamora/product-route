# Product Route · 产品路线判断 Skill

> **Product Route 是一个基于证据的 AI Skill，用来把产品想法路由到合适的推进模式。**

它不问「这个想法好不好」—— 它问「这个想法接下来应该怎么处理？」

---

## 一句话定位

把产品想法从「要不要做」的无限纠结中解放出来，用证据和规则回答一个更实际的问题：**现在应该用什么模式推进？**

---

## Quick Start

### 前置条件

- 在任何支持 Skill 机制的 AI Agent 中使用（Claude Code / Cursor / 自定义 Agent 等）
- 无需安装依赖、无需联网、无需 API Key

### 使用方式 A：作为 Skill 包安装（推荐）

将 `skills/product-route.zh-CN/` 目录安装为 Skill 包。

Agent 会在检测到产品想法评估、产品定位、MVP 边界、推进模式、风险分类等意图时自动触发。

### 使用方式 B：作为单文件 Prompt Bundle

将 `dist/product-route.zh-CN.bundle.md` 作为 System Prompt 或一次性 Prompt 注入。

适合不支持 Skill 目录结构的 Agent 环境。

### 使用方式 C：程序化调用

参考 `schemas/result.schema.json`，将输出 JSON 接入你的产品决策 pipeline。

## 9 种路由

| ID | 类型 | 说明 |
|---|---|---|
| R01 | 入口工具 | 低门槛工具，靠 SEO/分发获客，单个开发者可交付 |
| R02 | 货架服务 | 标准化 SaaS，用户自助购买，按需付费 |
| R03 | 内部基础设施 | 复用型内部工具，不直接面向外部用户 |
| R04 | 第三方封装 / 集成层 | 封装底层复杂服务，提供简化界面 |
| R05 | 销售型服务 | 需要销售介入、定制接入、合同谈判 |
| R06 | 内容或社区产品 | 以内容或社群为核心价值，而非工具功能 |
| R07 | 高风险基础设施，暂缓 | 涉及支付、认证、视频、合规等高风险，建议暂缓或窄化 |
| R08 | 只适合做功能，不适合独立产品 | 应作为现有产品的功能模块，而非独立项目 |
| R09 | 信息不足，先调研 | 关键维度缺乏证据，需要先收集信息再做判断 |

## 这个项目不是

- ❌ 创业点子打分器
- ❌ 商业计划书生成器
- ❌ 投资建议工具
- ❌ 法律 / 财务 / 合规顾问
- ❌ 「你的想法值多少钱」计算器

## 项目结构（For Maintainers）

| 目录 | 用途 |
|---|---|
| `skills/` | Skill 包 — 中文和英文运行包 |
| `dist/` | 单文件运行包 — 由 build-bundles.ts 生成 |
| `schemas/` | 结构化输出 — idea / evidence / result JSON Schema |
| `tests/` | 回归案例 — 期望路由断言 |
| `extensions/` | 未来 MCP / Provider 预留 — 核心 Skill 不依赖这些 |
| `src/` | 辅助脚本 — 构建和校验，不是核心规则 |

---

## 核心流程

```
Evidence → Score → Risk Gate → Route → MVP Boundary → Validation Action
```

1. 复述产品想法
2. 收集证据
3. 寻找反证
4. 12 维评分
5. 证据置信度惩罚
6. 风险门槛检查
7. 输出主路由 + 副路由
8. 定义 MVP 边界
9. 输出最低成本验证动作
10. 输出结构化 JSON
