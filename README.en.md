# Product Route · Product Idea Routing Skill

> **Product Route is an evidence-based AI skill that routes product ideas to the right execution mode.**

It doesn't ask "Is this a good idea?" — it asks "What should happen next with this idea?"

---

## One-Liner

Free yourself from the endless "should I build this" debate. Use evidence and rules to answer a more practical question: **What mode should I use to move this forward?**

---

## Quick Start

### Prerequisites

- Works in any AI Agent that supports the Skill mechanism (Claude Code, Cursor, custom agents, etc.)
- No dependencies, no internet required, no API keys

### Usage A: Install as Skill Package (Recommended)

Install the `skills/product-route.en/` directory as a Skill package.

The agent auto-triggers when it detects intent around product idea evaluation, product positioning, MVP boundaries, execution modes, risk classification, etc.

### Usage B: As a Single-File Prompt Bundle

Inject `dist/product-route.en.bundle.md` as a System Prompt or one-shot prompt.

For agent environments that don't support Skill directory structures.

### Usage C: Programmatic

Reference `schemas/result.schema.json` to integrate the output JSON into your product decision pipeline.

## 9 Routes

| ID | Name | Description |
|---|---|---|
| R01 | Entry Tool | Low-barrier tool, acquires via SEO/distribution, single dev deliverable |
| R02 | Shelf Service | Standardized SaaS, self-serve purchase, pay-as-you-go |
| R03 | Internal Infrastructure | Reusable internal tool, not directly facing external users |
| R04 | Third-party Wrapper / Integration Layer | Wraps complex underlying services, provides simplified interface |
| R05 | Sales-led Service | Requires sales involvement, custom integration, contract negotiation |
| R06 | Content / Community Product | Content or community as core value, not tool functionality |
| R07 | High-risk Infrastructure, Defer | Involves payments, auth, video, compliance; recommend narrowing or deferring |
| R08 | Feature Only, Not Standalone | Should be a feature module in an existing product, not standalone |
| R09 | Research First | Critical dimensions lack evidence; research before evaluation |

## Core Flow

```
Evidence → Score → Risk Gate → Route → MVP Boundary → Validation Action
```

1. Restate the product idea
2. Collect evidence
3. Find counter evidence
4. 12-dimension scoring
5. Evidence confidence penalties
6. Risk gate check
7. Output main route + secondary route
8. Define MVP boundary
9. Output minimum-cost validation actions
10. Output structured JSON
